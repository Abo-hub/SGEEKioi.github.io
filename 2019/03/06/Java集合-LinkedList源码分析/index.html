Java集合-LinkedList源码分析

发表于 2019-03-06 更新于 2020-03-07 分类于 集合 阅读次数：

简介

LinkedList实现了List和Deque接口。 实现所有可选列表操作，并允许所有元素（包括null ）。LinkedList底层使用的双向链表结构使插入和删除操作更加高效。注意，LinkedList是线程不安全的。如果多个线程同时访问链接列表，并且至少有一个线程在结构上修改列表，则必须在外部进行同步。可以使用Collections类中的synchronizedList方法(这最好在创建时完成，以防止意外的不同步访问列表)：

结构分析


上图就是LinkedList的结构。我们看到LinkedList类中有一个内部类Node

这个类就代表双向链表节点Node。这个类有三个属性

  • next：前驱结点
  • prev：后继节点
  • item：节点的值

方法

  • add(E e)：将指定的元素追加到此列表的末尾

  • addLast(E e): 将指定的元素追加到此列表的末尾。

这里我们看到add()、addLast()两个方法都是调用linkLast()这个方法。所以说add()方法每次插入的地方也是列表的末尾。

  • add(int index, E element)：在此列表中的指定位置插入指定的元素。

==linkBefore需要传入两个参数，一个插入的值，一个指定的node，所以这里有调用了node(index)方法去找到index对应的node==

  • addAll(Collection<? extends E> c)：按照指定集合的迭代器返回的顺序将指定集合中的所有元素追加到此列表的末尾。

  • addAll(int index, Collection<? extends E> c):将指定集合中的所有元素插入到此列表中，从指定的位置开始。

  • addFirst(E e) ：在该列表开头插入指定的元素。

根据位置取数据

  • get(int index)：返回此列表中指定位置的元素。

  • **获取头节点(index=0)数据方法

区别：
这四个获取头节点方法的区别在于对链表为空时的处理，是抛出异常还是返回null。 getFirst()、element() 方法将会在链表为空时，抛出异常:NoSuchElementException
而另外两个方法会在链表为空时返回null

  • 获取尾节点(index=-1)数据方法：

区别： getLast()方法在链表为空时会抛出NoSuchElementException异常；而peekLast()列表为空时返回null

根据对象得到索引

  • indexOf(Object o): 返回此列表中指定元素的第一次出现的索引，如果此列表不包含元素，则返回-1。

  • lastIndexOf(Object o): 返回此列表中指定元素的最后一次出现的索引，如果此列表不包含元素，则返回-1。

检查链表是否包含某元素的方法:

  • contains(Object o)： 如果此列表包含指定的元素，则返回 true 。
